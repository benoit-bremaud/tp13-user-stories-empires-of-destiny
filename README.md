# TP13 — User Stories: Empires of Destiny

Ce repository sert de support pour l'exercice de gestion de produit agile autour du projet fictif **Empires of Destiny**.

## Contexte pédagogique
L'objectif est de transformer une retranscription de réunion Product Owner en backlog exploitable et priorisé, jusqu'à la définition d'un MVP.

## Structure du repository
- `README.md` : contexte, mode de travail et conventions
- `docs/livrable.md` : document principal du rendu
- `docs/checklist-conformite.md` : checklist de conformité à l'énoncé
- `.github/pull_request_template.md` : template de PR pour revue structurée

## Objectif du livrable
Produire un rendu final (PDF) contenant :
- extraction des fonctionnalités
- EPICS
- User Stories
- critères d'acceptation + Definition of Done
- backlog priorisé
- MVP

## Organisation du travail
- 1 issue GitHub = 1 tâche majeure
- 1 branche dédiée par issue
- commits réguliers et atomiques
- PR finale liée à l'issue (`Closes #<numéro>`)
